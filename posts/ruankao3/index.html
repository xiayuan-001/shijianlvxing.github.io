第一章 信息化和信息系统【下】

软件工程

软件架构设计

  • 组织结构
  • 拓扑结构
架构风格【毒沟尽是蛆 数据列道过 虚拟释规则 仓库有版本 调回丞相层】

  • 独立构件

    • 进程通信
    • 事件驱动

  • 数据流

    • 批处理
    • 管道/过滤器

  • 虚拟机

    • 解释器
    • 基于规则

  • 仓库

    • 数据库
    • 黑板
    • 超文本

  • 调用/返回

    • 主程序\子程序
    • 面向对象
    • 层次结构

软件设计

  • 结构化设计

概要设计、详细设计；原则是“高内聚、低耦合”

  • 面向对象设计

封装、继承、多态

设计模式分类

类模式（静态关系） 、对象模式（动态关系）

创建型模式——创建对象

结构型模式——处理类或对象的组合

行为型模式——描述类和对象的交互及职责分配

软件测试

测试用例

  • 不运行程序
  • 文档——检查单
  • 代码——桌前检查【开发阅读】、代码走查【脑子模拟】、代码审查【审查小组】
  • 完全清楚内部结构和算法，用于“单元测试”
  • 主要方法： 控制流测试、 数据流测试、 程序变异测试
  • 常用技术：“逻辑覆盖”
  • 完全不考虑结构和算法
  • 用于“集成、系统、确认测试”
  • 常用技术：等价类划分”、“边界值分析”、”正交试验法”

测试类型

单元测试—-模块内（开发自己做）

集成测试—-模块间（QC做）

系统测试—-完整集成的系统（QC做）

确认测试

  • Alpha测试【开发环境，受控】
  • Beta测试【实际使用环境，不受控】
  • 验收测试【客户做】

配置性测试—-软件配置项与SRS一致

回归测试—-变更部分正确性

群集现象

经验表明，测试后程序中残存的错误数目与该程序中已发现的错误数据成正比

软件调试

  • 测试—-找出错误；在先；有预知结果；
  • 调试—-定位错误；在后；无法预计；

集成

表示集成（集成点–界面）

控制集成（功能集成、应用集成、集成点–应用逻辑）

数据集成（集成点–中间件）

统一数据的工作流组成

大多数要实施电子商务的企业

运维

口诀：就是鱼丸
  1. 纠错型（纠正错误）
  2. 适应型（适应改变）
  3. 预防型（修复潜在）
  4. 完善型（扩充改善）

新一代信息技术

物联网

通过信息传感设备（如射频识别RFID），按约定的协议，将任何物品与互联网相连接

两项关键技术

  • 传感器技术
  • 嵌入式技术

三层架构

  • 感知层
  • 网络层
  • 应用层

智慧城市

物联网在城市管理中综合应用

五个功能层
  • 物联感知层（通过传感器、监控摄像机、 GPS终端，进行信息采集、识别和监测）
  • 通信网络层（以互联网、电信网、广播电视网为基础，组成网络通信基础设施）
  • 计算与存储层（包括软件资源、计算资源和存储资源，提供数据存储和计算）
  • 数据与服务支撑层（利用SOA、云计算、大数据等技术，提供服务和共享资源）
  • 智慧应用层（智慧交通、智慧旅游等，提供整体的信息化应用和服务）
三个支撑体系
  • 安全保障体系
  • 标准规范体系
  • 建设和运营管理体系

云计算

primary 提示块标签

  • IaaS（基础设施即服务）：硬件、存储等变服务；
  • PaaS（平台即服务）：操作系统、数据库、 WEB应用等变服务；
  • SaaS（软件即服务）：应用软件、组件、工作流等变服务；

大数据

5V
  • 大量
  • 多样
  • 价值
  • 高速
  • 真实性

数据之和的价值远远大于各数据价值之和

四个环节
  • 数据采集（主要技术数据抽取工具 ETL )
  • 数据存储
  • 数据管理
  • 数据分析与挖掘

区块链

是一种去中心化的分布式账本数据库，是比特币的底层技术

六大特征
  • 去中心化—-共同执行程序，所有电脑关机才会停止
  • 开放性—-数据对所有人公开， 任何人都可以查询，信息高度透明
  • 去信任公开透明，无需信任也可进行交易，无法欺骗彼此
  • 自治性—-使对人的信任改成了对机器的信任，任何人为干预不起作用
  • 匿名性—-固定算法自动判断，交易对手无须通过公开身份
  • 信息不可篡改—-分布式存储， 每个人都有副本
六层模型
  • 数据层—-数据加密和时间戳等基础数据和基本算法；
  • 网络层—-分布式组网机制、数据传播机制和数据验证机制等；
  • 共识层—-主要封装网络节点的各类共识算法网络节点的各类共识算法；
  • 激励层—-主要包括经济激励的发行机制和分配机制等；
  • 合约层—-主要封装各类脚本、 算法和智能合约，是区块链可编程特性的基础；
  • 应用层—-封装了区块链的各种应用场景和案例。

移动互联

移动通信与互联网二者结合，不能完全替代桌面互联网

无线网络

无线局域网(WLAN)

无线广域网(WWAN)

无线城域网(WMAN)

无线个人网(WPAN)

无线AP接入点—-是无线网络的核心， 最大覆盖范围100-400米， 传输速率可高达11Mbps

  • 4G—-TD-LTE和FDD-LTE两种制式。速度为： 100Mbps
  • 5G—-理论上可在28GHz超高频段以1Gbps的速度，最长传送距离可达2公里。

商业BI

将组织中现有的数据转化为知识， 帮助组织做出明智的业务经营决策

四阶段

数据预处理、 建立数据模型、 数据分析、 数据展现

3个关键技术

数据仓库、 联机分析处理、 数据挖掘、数据备份和恢复等

实施6步骤
  1. 需求分析；
  2. 数据仓库建模；
  3. 数据抽取；
  4. 建立商业智能分析报表；
  5. 用户培训和数据模拟测试；
  6. 系统改进和完善

互联网+

“互联网+各个传统行业”

六大特征
  • 跨界融合
  • 创新驱动
  • 重塑结构
  • 尊重人性
  • 开放生态
  • 连接一切

信息技术应用

电子政务

应用模型
  1. G2G【政府对政府】
  2. G2B【政府对企业】
  3. G2C【政府向公众】
  4. G2E【政府对公务员】

电子商务

最重要的两个协议是： SSL和SET

  • B2B——企业对企业【阿里巴巴】
  • B2C——企业对消费者【京东、天猫】
  • C2C——消费者和消费者【淘宝、咸鱼】
  • B2A——企业和政府
  • O2O——线上购买,线下提货

两化融合

信息化进程和工业化进程不再相互独立进行，两者在技术、产品、管理等各个层面相互交融，不可分割

四方面融合

  • 信息化—工业化；

  • 信息资源—工业资源；

  • 信息技术–工业技术；

  • 虚拟经济—实体经济

实施“中国制造2025”，促进两化深度融合，需要制订“互联网+”行动计划， 智能制造是两化深度融合的切入点和主攻方向。信息物理系统（CPS , Cyber-Physical Systems） 是智能制造的核心

其他知识

信息系统监理

依法设立、具备相应资质、第三方的、受业主委托，依据监理合同、建设合同，对信息系统工程项目实施的监督管理

监理工作的四控+三管+一协调
  • 四控—-投资、进度、质量、变更
  • 三管—-合同、信息、安全
  • 一协调—-沟通协调

大型信息系统

以信息技术和通信技术为支撑， 规模庞大， 分布广阔，采用多级网络结构，跨越多个安全域，处理海量的， 复杂且形式多样的数据，提供多种类型应用的大系统

  • 特征： 规模庞大、跨地域性、网络结构复杂、业务种类多、数据量大、用户多

系统规划工具

  1. 过程/组织（P/O）矩阵—–组织对于某个过程是负责决策、还是主要涉及、还是不涉及；
  2. 资源/数据（R/D）矩阵—–某一类数据包含哪些资源的具体信息；
  3. 过程/数据类CU矩阵—–过程要使用某一方面的数据，还是会创建该数据；
  4. IPO图—–分析每个过程的输入和输出数据类；

综合布线

  1. 标准是： EIA/TIA 568A
  2. 跨越距离不超过3000米
  3. 建筑总面积不超过100万平方米
  4. 人口为50-5万人。
六个子系统
  • 建筑群子系统
  • 设备间子系统
  • 垂直干线子系统
  • 管理子系统
  • 水平子系统
  • 工作区子系统
